python调试技巧

发表于 2016-08-07 | |

在python代码的调试过程中如果我们需要知道执行到某函数时的调用栈信息，可在此函数内调用TraceFrame函数。如果我们想要知道栈帧中某个函数内的局部变量信息，对TraceFrame做出改进即可。

python通常被用来构建业务逻辑，变量含义通常也是直面业务逻辑的，不会像底层实现定义的变量含义那般抽象，而且python无需编译即可执行（准确说是程序猿对开发过程中的python编译不可感知）。所以绝大对数情况下python的调试使用打印即可，而无需使用pdb那般繁琐。而若直接打印某个我们自定义类的实例化对象，通常输出的是对象的类名及对象内存地址。这些输出对我们的调试工作并无多大帮助，所以只能跟近一步的打印出在对象之下我们所关心的属性值。但这样以来，调试的工作量就会大大增加。

我们可以通过改写类的repr方法，将我们最关心的对象信息在此返回，调试时直接打印对象即可。python解释器在执行print语句时将调用对象repr方法，并将此方法的返回字串打印出。在此同时解释下为何我们在print某个list或其它容器时，若此容器内部包含汉字字串，则输出并不会显示出汉字，而是一堆以”\x”打头的十六进制转义字串，其实际为汉字字串的内存值。发生这种情况的原因时print语句在遇到容器时将依次调用容器内对象的repr方法，并将返回的字串拼接最终打印出来。如果改写了str对象的repr方法则可直接输出汉字。可参见最前的演示代码。

添加打印调试时，如果添加打印的点较多，一方面输出容易弄混，另则在删除打印时则因命名不统一难以统一删除。上述vim插件，可辅助轻松高效的完成打印，尽可能的降低调试负担。

  1. vim normal 模式下输入快捷键 ;pf 调用插件
  2. 在打印语句中加入检索出的函数名
  3. 检索上下文，对同一函数内的其它打印语句做计数操作，并在本次添加的打印语句中加入打印计数计数值，用以区分不同流程下加入的打印
  4. 自动同步当前光标所在行的缩进层次，保证添加的打印语句缩进层次正确，并打印出函数的第一个参数
