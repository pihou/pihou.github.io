杂谈-讲故事与灵感

发表于 2017-02-04 | |

做事与讲故事

前段时间看到一篇关于腾讯某部门技术架构升级的文章,标题记得差不多是 从百万访问量到亿级访问量. 从文章的内容来看,并没见到多少高深技术的使用,也没见到多少唬人的术语,更多的是纯粹从业务方面来讲解遇到哪些难题, 做了哪些改进.当然,标题本身就很吓人,当别人看到标题本身的时候,至少我看到时,就立马想入非非了,认定肯定是些技术牛人,完成了牛逼的壮举. 但是文章通篇并没有多少花哨的东西,全是以义务为中心介绍的.不由得让自己多少有些意外.

要说上份工作,做游戏的那会儿,也是解决了不少业务难题的,虽是做手游,但是内容也算是游戏的核心(ai).但是在年终总结时怎么也bb不出像上文作者那么多的话,好多在做的过程中遇到的问题与难题,在完成之后便不以为然了.总结总是单写几句干货.或者说明自己不是一个会说故事的人,或者说明自己总结的不够,或是说明自己是个锤子眼中全是钉子.

灵感,创意到底是什么

最近好像顿悟了一点东西. 灵感与创造到底是什么? 多数人多数时间里,使用着日常熟悉或熟知的工具,技术,事物. 好像与灵感,创造,创意,这些高大上的词语毫无机缘, 平庸的是绝大多数. 但是或许所谓的灵感, 发明, 创造 只是将已有的东西,恰到好处的使用在合适的时机与合适的场景.机缘巧合,阴差阳错的推动了特定领域一丝丝的前进.或许这种描述在工程领域会更为适用些.
