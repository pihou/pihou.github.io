不懂装懂 MapReduce

发表于 2017-05-12 | |

工作中遇到一个需求, 原始数据只是记录好的仪表(比如水表,电表)在某一时刻的表盘数值,但是前端需要显示的是在每天,每周,每月的使用量. 我和同事说,这样的需求的话,我们就放到前端去算吧, 没想到同事说应该在后端算, python 的 MapReduce很快. 我听着感觉不太对劲,但是对MapReduce并不是很了解,也就没有说啥.

后续为此专门查阅了MapReduce,相关概念. 简单来说,MapReduce,是在做数据统计运算是的两个步骤, 一个是Map, 一个是Reduce, 和Python的内建函数相类似的, Map是在做数据统计前的加工处理, Reduce则是在计算最终需要的统计数据. 这些统计数据之前并无依赖性,可以放在分布式集群中,多机器并行计算. MapReduce,只是这种算法的思想核心而已. 快也仅仅只是因为算法的本身而已, 和实际程序执行效率并无关系. 重点是在与分布式,并发执行.

但是,同事的说法就很让人无语了,我们可不可以不要这么肤浅. 玩概念还玩的这么逗比的. 我个人是极为讨厌半懂不懂还装模作样的.为此自己还专门翻看了python Map Reduce的实现, 也不是什么黑魔法,和使用脚本去写实现也没啥大区别,只是在c层简单封装了一层仅此而已. 更何况仅仅Map Reduce并不能为完全代表MapReduce的全部思想与实现,单单是两个核心实现函数而已.
