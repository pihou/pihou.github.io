xss攻击案例分析

发表于 2017-05-30 | |

最近在阅读陈浩博客（coolshell.cn）web相关的文章，希望能拓宽自己的视野。

新浪微博的xss攻击一文对攻击发生的核心并未做阐明，可能耗子叔认为起因显而易见，无需赘述。但是对于自己而言，背后的原理并不是很清晰。所以在反复阅读原文及攻击源码的同时，在网上也搜集了其它资料与网友的观点。

原文给出的源码并不能说明任何问题。无非是是在新浪微博的页面下，之行了这些源码，调用了新浪的api而已，本身并没有任何神奇的地方。问题关键在于，攻击者是如何在新浪微博的页面下嵌入自己的js脚本的，也就是js脚本注入。

之前并没有任何的web安全相关经验，所以在查询sql注入及脚本注入相关讲解时，由于相对原理话，并不是很好理解。直到在知乎上都到一位网友给出的以下图片链接（因为攻击事件已经发生太久，所以无法动手复现了）

从图中，我们可以猜测，新浪微博的后端程序并没有对用户的访问的url做任何安全性检查，本来并不存在的页面并未返回404。而是将用户访问的页面直接使用到的返回的页面之中。按理说，这样最终生成的页面应该会渲染异常的，但是游览器本身的安全漏洞（双引号都没成对出现），将脚本下载了，由此攻击发生。

希望通过这个具体案例，能够帮助大家了解什么是xss攻击，以及背后的原理。web开发时过程一定记得对用户输入多做检查。
