python多线程浅析

发表于 2017-07-16 |

实际工作中对python多线程技术的使用少之又少。一直以来并未对python的多线程做过多深入。uwsgi在启动http服务时可以启动多进程和多线程。但是日常开发实际多数时候并不用关心多线程，一方面是uwsgi模块已经封装的很完善，另一方面则是因为http的无状态设计，在开放过程中涉及到使用全局变量的情形基本没有。

有次在项目的代码中看到了涉及多线程编程相关的代码，代码的核心意思是，将当前线程的某个状态变量保存到了一个类变量中，保存过程中，会以线程id作为键名保存，后续代码中将以此作为基础，获取相应数据。当时并没懂为什么要怎么操作。在心中扎了一根刺，现在终于理清了。实际这种做法也很赘余，直接在当前栈帧用个临时变量不就好了么。

python多线程，说穿只不过是将一个python对象通过c层线程api接口传入线程。新生成的线程将由系统分配空间负责其调用栈。线程间的同步问题，通过一个互斥锁之类的技术手段，将没有获取到gil的线程强制放入到系统的不可中断线程之类。由此保证同一时间只有一个线程会得到执行。GIL的释放只有在线程发生阻塞（接收sockt数据之类），或字节码在执行指定的条目数后才会释放gil，发生线程切换。

由此可见，python线程只不过是更多粒度的并发而已，和原生线程还是没法比的。但是也基本保证了在发生阻塞时，执行权的转移，毕竟最早并行计算的主要主要目标还是为了在服务IO时，将cpu空出来继续计算任务的。但同时也造成了多线程间无法并行计算在多核cpu上。

至于为什么python多线程间也需要同步技术（python语言开发级别），相信大家如果对python虚拟机理解的话，答案很浅显。
